## feature_10 — Кнопки выбора трека на карточках профессий

**Ветка:** `feature_10_orientation_choose_track`  
**MVP-требование:** US5 — «Каждая карточка профессии содержит кнопку выбора профессии как трек развития»

### Что было
После прохождения профориентационного теста результаты отображались как список карточек-ссылок (`<Link>`). Вся карточка вела на страницу профессии — выбрать трек можно было только оттуда.

### Что изменено

| Файл | Изменение |
|---|---|
| `src/components/OrientationQuiz.tsx` | Карточка профессии разбита на две кнопки: «Выбрать трек» и «Подробнее» |
| `messages/ru.json` | Добавлены ключи: `chooseTrack`, `viewAnalytics`, `trackChosen` |
| `messages/en.json` | То же для английского |

### Детали реализации
- Кнопка **«Выбрать трек»** вызывает `POST /api/track` прямо с экрана результатов, без перехода на страницу профессии. После выбора кнопка меняется на «Трек выбран» (зелёный, иконка ✓), остальные кнопки блокируются.
- Кнопка **«Подробнее»** — ссылка на `/professions/[slug]` (аналитика профессии).
- Прогресс-бар соответствия теперь виден прямо на карточке (зелёный ≥ 70%, голубой иначе).
- Сортировка по убыванию соответствия уже была в `topMatches()` — не менялась.

## feature_11 — Автогенерация карьерных целей при выборе трека

**Ветка:** `feature_11_goals_autogenerate`  
**MVP-требование:** US9 — «После прохождения профориентационного теста система генерирует долгосрочные карьерные цели»

### Что было
Карьерные цели генерировались только вручную — через кнопку «Сформировать» на дашборде. После прохождения теста и выбора трека цели не создавались автоматически.

### Что изменено

| Файл | Изменение |
|---|---|
| `src/app/api/track/route.ts` | После сохранения трека и roadmap — автоматически вызывает `generateGoals()` и сохраняет цели в БД |
| `src/components/OrientationQuiz.tsx` | После успешного выбора трека показывает уведомление «Карьерные цели сформированы» |
| `messages/ru.json` / `en.json` | Добавлен ключ `goalsAutoGenerated` |
| `tests/unit/goals-utils.test.ts` | +3 unit-теста: покрытие US9 (все поля, горизонт, все профессии) |
| `tests/e2e/journey.spec.ts` | +1 E2E-тест: выбор трека → цели видны на дашборде |

### Результат тестов
- Unit: **109/109** ✅
- E2E: **14/14** ✅

## feature_12 — Визуальный feedback обновления roadmap через чат

**Ветка:** `feature_12_chat_roadmap_update`  
**MVP-требование:** US8 — «Бот получает прогресс, обновляет roadmap и карьерный портрет, отправляет поддерживающие сообщения»

### Что было
Логика обновления roadmap (`detectCompletedSkills`, `motivationalBlock`, `buildCareerPortrait`) уже работала в API, но пользователь видел только мотивационный текст в ответе коуча. Не было чёткого сигнала, что шаги в roadmap реально изменились.

### Что изменено

| Файл | Изменение |
|---|---|
| `src/app/api/coach/route.ts` | Возвращает `updatedSteps: string[]` — список обновлённых навыков |
| `src/components/ChatUI.tsx` | Под ответом коуча показывает бейдж-ссылку «Карта обновлена: N навыков → открыть roadmap» |
| `messages/ru.json` / `en.json` | Добавлен ключ `roadmapUpdated` |
| `tests/unit/progress.test.ts` | +5 unit-тестов для US8: детекция нескольких навыков, граничные случаи мотивационного блока |
| `tests/e2e/journey.spec.ts` | +1 E2E-тест: бейдж `data-testid="roadmap-update-badge"` виден после сообщения о прогрессе |

### Результат тестов
- Unit: **114/114** ✅
- E2E: **15/15** ✅
